# Tool Selection Explanation

- source_material_reason: Input is treated as url for primary_website_evidence_or_style_reference.
- build_scope_reason: Build scope is component because the detected/requested UI scope maps to that Magic project type.
- magic_project_type_reason: Use Magic component to avoid generating the wrong UI surface.
- magic_creation_type_reason: Use get_inspired; from_scratch means no URL flow, get_inspired means URL-derived evidence or inspiration.
- website_goal_reason: Use drive_sales to pick the right conversion/workflow pattern.
- ui_ux_pro_max_mode_reason: UI/UX Pro Max must improve the UI beyond the source while respecting fidelity and source truth.
- motion_intensity_reason: Use standard motion after UI rebuild only, with purpose and reduced-motion fallback.
- qa_gate_reason: Design quality, visual parity, accessibility, responsiveness, and motion purpose gates must approve the output.

# Visual Acceptance Criteria
## Hospital Online Consultation Portal

| ID | Criterion | Pass Condition |
|----|-----------|---------------|
| VAC-001 | Login page shows hospital logo, email, password, login button | All elements visible, password masked |
| VAC-002 | Login split-screen composition balanced on desktop | Form panel 45-50%, illustration 50-55% |
| VAC-003 | Sidebar nav shows all 6 navigation items | All links visible with active state |
| VAC-004 | Dashboard shows 4 metric cards in a row | Teal numbers, gray labels, white cards |
| VAC-005 | Dashboard metric numbers update after booking | Live counter reflects state |
| VAC-006 | Dashboard action cards show 4 items in 2x2 grid | Open button teal, card white rounded |
| VAC-007 | Book Consultation form has 4-col grid for fields | 2 rows of 4 fields + full-width textarea |
| VAC-008 | All form inputs have visible labels | Label above each input, htmlFor linked |
| VAC-009 | My Consultations shows filter row + card list | Search + status + dept filter visible |
| VAC-010 | Consultation cards show patient name, dept, doctor, mode, date, status badge, update button | All fields visible |
| VAC-011 | Status badges are color-coded | Teal=Scheduled, Green=Completed, Amber=Requested |
| VAC-012 | Prescription page shows Available/Pending based on status | Teal badge for Available, amber for Pending |
| VAC-013 | Sign Out page shows confirmation card with red button | Checkmark icon, confirmation text, red CTA |
| VAC-014 | After sign out, app returns to login | URL redirects to / |
| VAC-015 | Demo disclaimer visible on login + throughout app | Demo Patient Portal badge in TopHeader |
| VAC-016 | No content overflow or horizontal scroll on 1280px+ | Layout fits within viewport |
| VAC-017 | TopHeader teal gradient visible on all authenticated pages | Gradient from #2BB5A0 |
| VAC-018 | Focus rings visible on all interactive elements | teal focus ring on tab |

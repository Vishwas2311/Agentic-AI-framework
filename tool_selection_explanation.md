# Tool Selection Explanation

- source_material_reason: Input is treated as uploaded_image for primary_visual_evidence.
- build_scope_reason: Build scope is website because the detected/requested UI scope maps to that Magic project type.
- magic_project_type_reason: Use Magic website to avoid generating the wrong UI surface.
- magic_creation_type_reason: Use from_scratch; from_scratch means no URL flow, get_inspired means URL-derived evidence or inspiration.
- website_goal_reason: Use custom to pick the right conversion/workflow pattern.
- ui_ux_pro_max_mode_reason: UI/UX Pro Max must improve the UI beyond the source while respecting fidelity and source truth.
- motion_intensity_reason: Use subtle motion after UI rebuild only, with purpose and reduced-motion fallback.
- qa_gate_reason: Design quality, visual parity, accessibility, responsiveness, and motion purpose gates must approve the output.

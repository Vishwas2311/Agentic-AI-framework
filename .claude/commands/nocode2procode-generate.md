# /nocode2procode-generate

Generate backend, frontend, database, API, tests, deployment files, observability, and docs from the canonical app spec.

## Required Behavior

1. Require `canonical_app_spec.json` or build it first.
2. Use deterministic templates before agent repair.
3. Use design strategy, visual lock, domain pack, component pattern, and quality gates for frontend generation.
4. Produce `generated_app/` and `generated_file_manifest.json`.
5. Do not skip the planning path; generation is downstream of extraction, source truth, and canonical spec creation.
